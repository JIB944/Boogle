# Pruebas Manuales - Boogle

## Casos de prueba

### 1. Inicio del juego
- [ ] Modal de inicio se muestra al cargar la página
- [ ] No se puede comenzar sin ingresar nombre
- [ ] Validación: nombre mínimo 3 letras
- [ ] Validación: nombre alfanumérico
- [ ] Selector de tiempo muestra 3 opciones (1, 2, 3 minutos)

### 2. Generación del tablero
- [ ] Se generan 16 letras aleatorias
- [ ] Las letras son variadas (no todas iguales)
- [ ] El tablero se muestra correctamente en grid 4x4

### 3. Temporizador
- [ ] El temporizador inicia al comenzar el juego
- [ ] El contador disminuye cada segundo
- [ ] El temporizador se actualiza visualmente
- [ ] Al llegar a 0, se detiene el juego
- [ ] Modal de fin de juego se muestra

### 4. Selección de letras
- [ ] Se puede seleccionar una letra haciendo click
- [ ] Solo se pueden seleccionar letras contiguas
- [ ] No se puede repetir la misma letra en una palabra
- [ ] La palabra se muestra mientras se forma
- [ ] El botón limpiar funciona correctamente

### 5. Validación de palabras
- [ ] No se aceptan palabras menores a 3 letras
- [ ] No se aceptan palabras ya encontradas
- [ ] Palabras válidas suman puntos
- [ ] Palabras inválidas restan puntos
- [ ] Se muestra feedback al usuario

### 6. Puntuación
- [ ] 3-4 letras: 1 punto
- [ ] 5 letras: 2 puntos
- [ ] 6 letras: 3 puntos
- [ ] 7 letras: 5 puntos
- [ ] 8+ letras: 11 puntos
- [ ] El puntaje se actualiza en tiempo real

### 7. Lista de palabras
- [ ] Las palabras encontradas aparecen en la lista
- [ ] Se muestra el puntaje de cada palabra

### 8. LocalStorage y Ranking
- [ ] La partida se guarda al finalizar
- [ ] El ranking se puede visualizar
- [ ] El ranking se ordena por puntaje
- [ ] El ranking se puede ordenar por fecha

### 9. Contacto
- [ ] Validación de nombre alfanumérico
- [ ] Validación de email válido
- [ ] Validación de mensaje (más de 5 caracteres)
- [ ] Se abre el cliente de email al enviar

### 10. Responsive
- [ ] El juego se ve bien en desktop (1920px)
- [ ] El juego se ve bien en tablet (768px)
- [ ] El juego se ve bien en móvil (375px)

### 11. Accesibilidad
- [ ] Se puede navegar con teclado
- [ ] Los elementos interactivos tienen foco visible
- [ ] Los atributos ARIA están presentes
